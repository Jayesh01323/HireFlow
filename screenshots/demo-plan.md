# Demo Recording Plan

## Overview

This document outlines the plan for creating demo recordings and walkthroughs for HireFlow AI.

## Recording Strategy

### Demo Types

1. **Feature Walkthrough** (2-3 minutes)
   - Quick overview of key features
   - Fast-paced, engaging
   - Background music optional

2. **Tutorial Videos** (5-10 minutes)
   - Step-by-step instructions
   - Detailed explanations
   - Screen recording with voiceover

3. **GIF Demonstrations** (10-30 seconds)
   - Specific feature highlights
   - Looping animations
   - No audio required

## Recording Guidelines

### Technical Specifications

- **Resolution**: 1920x1080 (1080p)
- **Frame Rate**: 30 fps (standard), 60 fps (smooth)
- **Format**: MP4 (video), GIF (animations)
- **Bitrate**: 5-10 Mbps (video)
- **Audio**: 44.1 kHz, stereo (if applicable)

### Content Guidelines

- **Clean Environment**: Close unnecessary tabs/apps
- **Sample Data**: Use realistic, professional data
- **Consistent Speed**: Smooth, deliberate movements
- **Clear Audio**: Crisp voiceover, no background noise
- **Professional Appearance**: Clean, organized interface

### Script Templates

#### Feature Walkthrough Script

```
Intro (5-10 seconds)
- Welcome to HireFlow AI
- Brief overview of what we'll see

Feature Demo (60-90 seconds)
- Show key feature
- Demonstrate main functionality
- Highlight benefits

Outro (5-10 seconds)
- Call to action
- Where to learn more
```

#### Tutorial Script

```
Introduction (15-30 seconds)
- What we'll learn
- Prerequisites
- Expected outcome

Step-by-Step (3-5 minutes)
- Clear instructions
- Visual demonstrations
- Tips and best practices

Conclusion (15-30 seconds)
- Summary
- Next steps
- Additional resources
```

## Recording Checklist

### Pre-Recording

- [ ] Test recording software
- [ ] Prepare sample data
- [ ] Clean desktop environment
- [ ] Test audio equipment
- [ ] Practice script/talking points
- [ ] Check lighting (if recording face)
- [ ] Ensure stable internet connection

### During Recording

- [ ] Start with clean slate
- [ ] Use consistent mouse speed
- [ ] Speak clearly and confidently
- [ ] Avoid filler words (um, uh, like)
- [ ] Keep movements deliberate
- [ ] Highlight key areas
- [ ] Maintain consistent pace

### Post-Recording

- [ ] Review recording for quality
- [ ] Edit out mistakes/long pauses
- [ ] Add intro/outro if needed
- [ ] Add captions/subtitles
- [ ] Optimize file size
- [ ] Test on different platforms
- [ ] Upload to hosting platform

## File Organization

### Folder Structure

```
media/
├── videos/
│   ├── walkthroughs/
│   ├── tutorials/
│   └── shorts/
├── gifs/
│   ├── features/
│   └── animations/
└── thumbnails/
```

### Naming Conventions

- **Videos**: `feature-name-version-date.mp4`
- **GIFs**: `feature-name-action.gif`
- **Thumbnails**: `feature-name-thumb.png`

Example: `resume-parser-upload-v1.0-2024-06-20.mp4`
